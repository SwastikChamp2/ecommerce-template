# Environment Variables Setup Guide

This guide will help you configure all the necessary environment variables for the Scribbly e-commerce application.

## Quick Start

1. **Copy the example file:**
   ```bash
   cp .env.example .env
   ```

2. **Fill in your actual values** in the `.env` file (see sections below)

3. **Never commit `.env`** - add it to `.gitignore`

---

## Environment Variables Reference

### 🔥 Firebase Configuration

Firebase is used for authentication, database (Firestore), and storage.

#### How to Get Firebase Credentials:

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Create a new project or select an existing one
3. Go to **Project Settings** (gear icon)
4. Under **Your apps**, select or create a web app
5. Copy the configuration values

#### Variables:

| Variable | Example | Description |
|----------|---------|-------------|
| `REACT_APP_FIREBASE_API_KEY` | `AIzaSyD...` | API key for Firebase project |
| `REACT_APP_FIREBASE_AUTH_DOMAIN` | `Scribbly.firebaseapp.com` | Firebase authentication domain |
| `REACT_APP_FIREBASE_PROJECT_ID` | `Scribbly-12345` | Your Firebase project ID |
| `REACT_APP_FIREBASE_STORAGE_BUCKET` | `Scribbly-12345.appspot.com` | Cloud Storage bucket URL |
| `REACT_APP_FIREBASE_MESSAGING_SENDER_ID` | `123456789` | Messaging sender ID |
| `REACT_APP_FIREBASE_APP_ID` | `1:123456789:web:abc...` | Firebase app ID |

**Example:**
```env
REACT_APP_FIREBASE_API_KEY=AIzaSyDxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
REACT_APP_FIREBASE_AUTH_DOMAIN=Scribbly.firebaseapp.com
REACT_APP_FIREBASE_PROJECT_ID=Scribbly-project
REACT_APP_FIREBASE_STORAGE_BUCKET=Scribbly-project.appspot.com
REACT_APP_FIREBASE_MESSAGING_SENDER_ID=987654321
REACT_APP_FIREBASE_APP_ID=1:987654321:web:xxxxxxxxxxxxx
```

### 📧 Email Configuration (Resend API)

Email functionality is handled through [Resend](https://resend.com) API.

#### How to Set Up Email:

1. Go to [Resend](https://resend.com) and sign up
2. Create an API key (starts with `re_`)
3. Verify your domain for better deliverability
4. Use a verified email address for `REACT_APP_EMAIL_FROM`

#### Variables:

| Variable | Example | Description |
|----------|---------|-------------|
| `REACT_APP_EMAIL_ENABLED` | `true` or `false` | Enable/disable email functionality |
| `REACT_APP_EMAIL_FROM` | `orders@yourdomain.com` | Email address for sending messages |
| `REACT_APP_SUPPORT_EMAIL` | `support@yourdomain.com` | Support email address |
| `REACT_APP_RESEND_API_KEY` | `re_xxx...` | Resend API key (for local development) |

**Example:**
```env
REACT_APP_EMAIL_ENABLED=true
REACT_APP_EMAIL_FROM=orders@yourdomain.com
REACT_APP_SUPPORT_EMAIL=support@yourdomain.com
REACT_APP_RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

#### Important Notes:

- **Local Development:** Use `REACT_APP_RESEND_API_KEY` in your `.env` file
- **Cloudflare Pages:** Set `RESEND_API_KEY` (without `REACT_APP_` prefix) in the Cloudflare Pages dashboard
- Email is **optional** - set `REACT_APP_EMAIL_ENABLED=false` to disable
- The email service will log warnings if not properly configured, but won't crash the app

### ⚙️ Standard Environment Variables

| Variable | Possible Values | Description |
|----------|-----------------|-------------|
| `NODE_ENV` | `development`, `production`, `test` | Specifies the application environment |
| `PUBLIC_URL` | `/` or `/subdirectory/` | Used if app is in a subdirectory |

**Default Values:**
```env
NODE_ENV=development
PUBLIC_URL=/
```

## Deployment Guides

### 🔧 Local Development

1. Create `.env` file:
   ```bash
   cp .env.example .env
   ```

2. Fill in all Firebase credentials

3. Add Resend API key if you want email functionality:
   ```env
   REACT_APP_RESEND_API_KEY=re_xxxxx
   ```

4. Install dependencies and start:
   ```bash
   npm install
   npm start
   ```

### ☁️ Cloudflare Pages

1. Connect your GitHub repository to Cloudflare Pages
2. Go to **Settings** > **Environment variables**
3. Add the following:
   - All `REACT_APP_*` variables (same as local)
   - `RESEND_API_KEY` (for server-side email sending)

**Architecture:**
- Client-side variables: `REACT_APP_*` prefix
- Server-side (Cloudflare Functions): Plain variable names (e.g., `RESEND_API_KEY`)

### 🚀 Other Deployment Platforms

**Vercel:**
- Go to **Settings** > **Environment Variables**
- Add all variables as above

**Netlify:**
- Go to **Site settings** > **Build & deploy** > **Environment**
- Add all variables

## Troubleshooting

### ❌ "Firebase not initialized" Error

**Solution:** Check that all Firebase variables are correctly set in `.env`:
```bash
cat .env | grep REACT_APP_FIREBASE
```

### ❌ "Email not working" Error

**Checklist:**
- [ ] `REACT_APP_EMAIL_ENABLED=true` is set
- [ ] `REACT_APP_EMAIL_FROM` is a verified email in Resend
- [ ] `REACT_APP_RESEND_API_KEY` is correct (starts with `re_`)
- [ ] Check browser console for detailed error messages

**For Cloudflare Pages:**
- [ ] Make sure `RESEND_API_KEY` is set (without `REACT_APP_` prefix)
- [ ] Check Cloudflare Function logs for errors

### ❌ "Blank .env Variables"

**Solution:** Copy values from Firebase Console and Resend dashboard:
1. Don't leave empty values
2. Replace all `your_*_here` placeholders with real values

## Security Best Practices

✅ **DO:**
- Use `.env.example` as a template (commit this file)
- Never commit `.env` with real values
- Rotate API keys regularly
- Use separate keys for development and production

❌ **DON'T:**
- Commit `.env` file to version control
- Share API keys in chat or email
- Use production keys in development
- Push Firebase credentials to GitHub
